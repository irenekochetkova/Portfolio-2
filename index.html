Homemade Food

The main idea of this website is to unite chefs and customers. The chefs can sell their own homemade food and customer can order it. If the users want to sell their own homemade food they should create an account as a chef and choose an option "chef". If the users want to order homemade food they choose an option "customer".

Languages and Technologies:

Ruby on Rails, VueJs, JavaScript, HTML, CSS, Bootstrap and jQuery

Project Homemade food

Home page about

  • Every user can see Home page with descriptions about service of this site and all chefs that provide homemade food through this site.
  • The user can use the button “login” on the home-page that is implemented with the Modal plugin of bootstrap or login through the button in the navbar, that leads to the login page.

Authentication registration

  • Users can create an account (sing up), edit and delete. Also they can change a password.
  • The validation is implemented when the user signs up or logins.
  • Chef`s account differs customer`s account by the access to the actions. The chefs have the rights to create new dish, and the customers have the rights to see their orders and shopping cart.

User Logged in as a chef

  • Every logged user can see Menu page with dishes from all chefs.
  • Every logged user can use the filter to search dish by the name.
  • Every user can use the filter to search dish by cuisine.
  • Chefs can use the filter to search only their own dishes.
  • Every user can press the button on card of every dish “More info” and get more information about current dish.
    • ...
  • If Chefs press this button they can see the rights to edit and delete current own dish.
  • Chefs don’t have the right to edit and delete dishes, that were not created by them.
  • The chefs have the rights to create new dish and they can get Dihes-new Page through their account.
  • The chefs have the rights to edit their own dishes and they can get Dihes-edit Page on Menu Page through the current dish.
  • The validation is implemented when the user presses button “submit”.

User Logged in as a customer

  • Every logged user can see Menu page with dishes from all chefs.
  • Every logged user can use the filter to search dish by the name.
  • Every user can use the filter to search dish by cuisine.
  • Every user can press the button on card of every dish “More info” and get more information about current dish.
  • If customers press this button it will allow to add the dish to their shopping cart with required amount.
  • Customers can not see buttons to edit and delete dish.
  • Only customers can see icon-button carted shop to go to the page shopping cart and button to go to their orders.
  • Customer-users can see the amount of items in the shopping cart.
  • Customer-users can see your Shopping cart page where every carted dish includes price, quantity and subtotal sum.
  • Customer-users can delete cartedDish.
  • Customer-users can see total sum their purchases.
  • Customer-users can create an order if press button "checkout".
  • Customer-users can see their purchases on the Orders page.
  • Customer-users can cancel their orders.
